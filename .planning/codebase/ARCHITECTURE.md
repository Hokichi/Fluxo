# Codebase Map: Architecture

Generated: 2026-06-15

## Summary

Fluxo uses a layered WPF architecture: domain contracts in `Fluxo.Core`, EF Core persistence in `Fluxo.Data`, business/data services in `Fluxo.Services` and the app project, shared UI primitives in `Fluxo.Resources`, and WPF windows/pages/view models in `Fluxo`.

## Layers

- Domain layer: `Fluxo.Core/Entities`, `Fluxo.Core/DTO`, `Fluxo.Core/Enums`, `Fluxo.Core/Interfaces`, `Fluxo.Core/Budgeting`.
- Persistence layer: `Fluxo.Data/Context`, `Fluxo.Data/Repositories`, `Fluxo.Data/Operations`, `Fluxo.Data/UnitOfWork.cs`.
- Service layer: `Fluxo.Services/Persistence`, `Fluxo.Services/Backups`, `Fluxo.Services/Logging`, plus app-specific services under `Fluxo/Services`.
- Presentation layer: `Fluxo/ViewModels`, `Fluxo/Views`, `Fluxo/Extensions/ServiceCollectionExtensions.cs`.
- Resource/control layer: `Fluxo.Resources/Resources`, `Fluxo.Resources/CustomControls`, `Fluxo.Resources/Components`, `Fluxo.Resources/Converters`.
- Packaging layer: `Fluxo.Installer`, `Fluxo.Installer.Msi`, `Fluxo.Installer.Bundle`.

## Composition Root

- Main composition starts in `Fluxo/App.xaml.cs`.
- Services are registered by chaining `AddFluxoData`, `AddFluxoPresentation`, and `AddUIData`.
- Data services and repositories are registered in `Fluxo.Data/Extensions/ServiceCollectionExtensions.cs`.
- Presentation services, view models, windows, pages, and popups are registered in `Fluxo/Extensions/ServiceCollectionExtensions.cs`.
- AutoMapper profiles are combined from `Fluxo.Services/Mappings/EntityDtoProfile.cs` and `Fluxo/Mappings/DtoViewModelProfile.cs`.

## Startup Flow

- `Fluxo/App.xaml.cs` registers global exception handlers in the constructor.
- Startup checks single-instance state through `Fluxo/Infrastructure/SingleInstance/SingleInstanceStartupPolicy.cs`.
- Startup initializes logging, ensures database directory, backs up database, runs migration, syncs budget allocation period, then loads UI.
- First run opens `Fluxo/Views/Shell/Wizard/QuickSetupWizard.xaml`.
- Normal run opens `Fluxo/Views/Shell/Main/MainWindow.xaml` or tray mode.

## Data Flow

- View models call service interfaces or `IDataOperationRunner`.
- `IDataOperationRunner` creates a scoped data operation through `IDataOperationScopeFactory`.
- `DataOperationScope` exposes `IUnitOfWork`.
- Unit of work exposes typed repositories and commits through `SaveChangesAsync`.
- Repositories use `FluxoDbContext` and EF Core queries.
- AutoMapper moves data between entities, DTOs, and view models.

## Persistence Pattern

- `Fluxo.Data/Repositories/Repository.cs` is the generic base repository.
- Specific repositories add query methods and filters, for example `Fluxo.Data/Repositories/ExpenseLogRepository.cs`.
- Base repository prefers `AsNoTracking` reads and handles tracked update/remove conflicts by copying into tracked instances.
- `FluxoDbContext` configures tables, relationships, numeric column types, and reference auto-includes.
- `BudgetAllocation` has a singleton key enforced by unique index.

## UI Pattern

- Main shell is `Fluxo/Views/Shell/Main/MainWindow.xaml`.
- Main shell hosts pages: dashboard, analytics, calendar, ledger.
- Main view model is `Fluxo/ViewModels/Shell/Main/MainVM.cs`.
- Dashboard composes panels such as budget allocation, spending allowance, notifications, saving goals, and upcoming events.
- Popups are concrete WPF views under `Fluxo/Views/Popups`.
- MVVM uses `ObservableRecipient`, generated observable properties, and `WeakReferenceMessenger`.

## Installer Architecture

- Managed bootstrapper entry point: `Fluxo.Installer/Program.cs`.
- Bootstrapper logic: `Fluxo.Installer/BootstrapperEntry.cs`.
- Installer view model: `Fluxo.Installer/ViewModels/InstallerViewModel.cs`.
- MSI package authoring: `Fluxo.Installer.Msi/*.wxs`.
- Bundle authoring: `Fluxo.Installer.Bundle/Bundle.wxs`.
- Bundle includes managed bootstrapper payloads generated by an MSBuild target.

## Error Handling

- Data operations wrap failures in `DataOperationException` in `Fluxo.Data/Operations/DataOperationRunner.cs`.
- App startup catches broad failures, logs with `FluxoLogManager`, and shows a dialog.
- Global dispatcher, app-domain, and unobserved task handlers log unhandled failures in `Fluxo/App.xaml.cs`.

## Key Entry Points

- App entry: `Fluxo/App.xaml.cs`.
- Installer entry: `Fluxo.Installer/Program.cs`.
- Data context: `Fluxo.Data/Context/FluxoDbContext.cs`.
- Dependency registration: `Fluxo/Extensions/ServiceCollectionExtensions.cs`.
- Data registration: `Fluxo.Data/Extensions/ServiceCollectionExtensions.cs`.
- Release automation: `.github/workflows/build-on-final-commit.yml`.
